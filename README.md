## 5. Mathematische Gewichtung (Der Boost)
Um sicherzustellen, dass kritische Filter-Treffer trotz Millionen von Domains im Export landen, nutzt die Pipeline eine mathematische Aufwertung:

$$Rank_{final} = \max(1, Rank_{initial} - Boost)$$

Jeder Treffer aus einer Filterliste erhaelt einen strategischen Bonus von **950'000** Plaetzen. Dadurch ruecken blockierte Domains massiv nach vorne und verdraengen weniger relevante Inhalte.

---

## 6. Die technologische Grenze: Tor-Netzwerk
Die **X-iNet Multi-Format Filter Pipeline** arbeitet hocheffektiv auf DNS-Ebene. 

> **Wichtiger Hinweis:** Das **Tor-Netzwerk (Onion Routing)** stellt eine systemische Schwachstelle fuer DNS-basierte Filter dar. Da Tor den Datenverkehr in einem verschluesselten Tunnel direkt zu Einstiegsknoten leitet und die Namensaufloesung erst am Ende der Kette erfolgt, kann ein lokaler DNS-Filter diesen Verkehr nicht einsehen oder blockieren. Eltern sollten fuer diesen speziellen Fall zusaetzliche Schutzmassnahmen auf den Endgeraeten in Betracht ziehen.

## 7. Installation unter GitHub Codespaces
Die Pipeline ist fuer den Betrieb in **GitHub Codespaces** optimiert, um eine sofort einsatzbereite Entwicklungsumgebung zu bieten.

1.  **Codespace starten:** Oeffnen Sie das Repository in GitHub und klicken Sie auf `Code` -> `Codespaces` -> `Create codespace on main`.
2.  **Abhaengigkeiten:** Das System erkennt die Anforderungen automatisch. Falls nicht, fuehren Sie `pip install requests` aus.
3.  **Pipeline ausfuehren:** Starten Sie den Prozess mit `python main.py`.
4.  **Ergebnis:** Alle 10 Export-Formate werden im Ordner `/output` bereitgestellt.
